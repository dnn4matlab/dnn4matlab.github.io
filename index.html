Deep Neural Networks for Matlab

Overview

dnn4matlab provides a fast CPU and GPU (CUDA) code to train large neural networks. Code is developed in Matlab, and contains CUDA bindings. With this code we deliver trained models on ImageNet dataset, which gives top-5 accuracy 13% on ImageNet validation set. Main focus of this code is to give reasonable computer vision models to people having no experience in field. Feel free to use, modify and sell this code for any purpose (commercial, or academic). Please just mention at the end where you get it from.

To get started just checkout our repository:

$ git clone https://github.com/dnn4matlab/code

Evaluate ImageNet trained model on your data

File eval.m shows how to evaluate trained model on your data. It first downloads trained model from a server, and stores it into /models directory (this is done only once). 

load_imagenet_model();
predictions = test("data/doggy.png");% You can use an url.
predictions[1:10]
Evaluating a single image is quite slow in compare to evaluating many images in the same time. We recommend to store 128 images of the size 224 x 224 in a single array. Code eval128.m shows how to evaluate 128 images simultaneously. This might take a second at first, because we download entire imagenet validation set (50k images) from our servers.

load_imagenet_model();
imgs = get_images()
predictions = test(imgs);
for i = 1, 128
  imagesc(squeeze(imgs[i, :, :, :]))
  predictions[i, 1]
  waitforbuttonpress
end

This code displays best prediction for every out of 128 examples.

Train your own model

Files in a directory plans describe various neural networks architecture. Feel free to define your own architectures, and try how they work. We provide reasonable architectures for MNIST, and ImageNet models.

MNIST

File training.m trains MNIST model. It first downloads MNIST dataset.

json = ParseJSON('plans/mnist.txt');
Plan(json);
Run();

Models are stored in directory /models. After training MNIST model is available by calling:

load_model("models/mnist.m");

ImageNet

Training dataset for ImageNet is much bigger than MNIST. To get it you have to register on Image-Net website.

Internals

Entire model is a global variable called Plan. To access it call

global Plan

All layers are stored in a directory /layers, and code to verify their correctness is a file GC.m (gradient checker). To test new layer, please extend file tests.m, and run GC.m.
